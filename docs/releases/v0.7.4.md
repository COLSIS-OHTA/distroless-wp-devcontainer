# v0.7.4

## 概要

DevContainer 向けの **WP-CLI（`compose-wp`）** と **All-in-One WP Migration** の運用を改善しました。

## 追加・変更

### DevContainer

- Node.js **22.22.2**（`node:22.22.2-bookworm`）
- `compose-wp` — 実行中 `wp-cli` コンテナへ `docker exec`（ワークスペース版スクリプトに委譲）
- `setup-docker-access.sh` — Docker ソケット権限の同期
- `post-start.sh` — 起動時セットアップ

### Docker Compose

- `ai1wm-backups` ボリュームを `php` / `wp-cli` にマウント
- 先頭に `name: ${COMPOSE_PROJECT_NAME:-wp}` を追加（プロジェクト名の明示）
- `COMPOSE_PROJECT_NAME` 変更時は **Rebuild Container** と `docker compose -p wp down` が必要（ドキュメント追記）
- `COMPOSE_PROJECT_NAME` は **小文字のみ**（大文字混在は `docker compose config` 失敗 → Dev Container 起動エラー）

### ドキュメント

- `docs/devcontainer.md` — トラブルシューティング（古いイメージ、Docker 権限、プロジェクト名）

## 使い方

1. 最新の `main` を取得し、**Dev Containers: Rebuild Container** を実行
2. 確認:

```bash
docker compose version
compose-wp plugin list
```

All-in-One WP Migration の復元:

```bash
cp backup.wpress www/content/ai1wm-backups/
compose-wp ai1wm restore backup.wpress
```

## 含まれるコミット（v0.7.3 以降）

- DevContainer: Docker ソケット権限、`compose-wp` 修正、Node 22.22.2
- Compose: `ai1wm-backups` マウント、プロジェクト名 `name:` フィールド
